## RunGoal Admin - 后台管理系统功能规划

### 定位

面向运营者（你自己）的轻量管理后台，核心用途是：看数据、管用户、调配置、做运营。不搞大而全的后台框架，够用、好用就行。

### 技术选型建议

| 方向 | 选型 | 理由 |
|------|------|------|
| 框架 | React + Vite + TypeScript | 和 C 端共享技术栈，可复用 shared 包 |
| UI 库 | Ant Design 5 | 后台标配，组件全、开箱即用 |
| 图表 | ECharts (同 C 端) | 统一图表方案 |
| 部署 | 独立端口或子路径，和 C 端共用后端 API | 复用同一套 Rust/Axum 服务，加 admin 权限中间件 |

可以放在同一个 monorepo 里作为 `apps/admin`，和 `apps/web` 平级。

---

### 功能模块

#### 1. 数据仪表盘（Dashboard）

首页一屏看全 App 运营状态。

**核心指标卡片**（顶部一排）：
- 总用户数 / 今日新增用户
- 总跑步次数 / 今日跑步次数
- 总跑量 (km) / 今日跑量
- 活跃用户数（近 7 天有跑步记录的用户）

**图表区域**：
- 近 30 天每日跑步次数折线图
- 近 30 天每日新增用户折线图
- 用户活跃度分布（按周跑次数分桶：0次、1-2次、3-5次、5次以上）
- 今日跑步时段分布（24小时柱状图，看出大家喜欢什么时候跑）

**快捷入口**：
- 最近的跑步记录列表（最新 10 条）
- 最近注册用户列表（最新 10 个）

---

#### 2. 用户管理

**用户列表**：
- 表格：昵称、手机号（中间四位打码）、注册时间、总跑量、总次数、最近跑步时间、状态
- 支持搜索（昵称/手机号）、筛选（注册时间范围、跑量范围、状态）
- 支持排序（按注册时间、总跑量、最近活跃）
- 分页

**用户详情**：
- 基础信息：昵称、头像、体重、身高、注册时间
- 跑步统计：总跑量、总次数、总时长、最长单次距离、最佳配速
- 跑步记录时间线（该用户的所有跑步记录，按时间倒序）
- 目标列表（该用户的所有目标及当前进度）
- 成就解锁情况（已解锁 / 全部，解锁率）
- 挑战列表（进行中 / 已完成 / 已失败）

**操作**：
- 编辑用户资料（改昵称等）
- 禁用/启用账号
- 重置密码
- 查看用户设备/登录日志（如果后续加登录日志表的话）

---

#### 3. 跑步记录管理

**记录列表**：
- 表格：用户昵称、距离(km)、时长、配速、来源(手动/GPS)、感受评分、跑步时间
- 支持筛选：日期范围、用户、来源类型、距离范围
- 支持排序：按时间、距离、配速
- 分页

**记录详情**：
- 完整数据展示：距离、时长、配速、卡路里、感受、备注、天气
- GPS 轨迹（如果有的话，在地图上展示轨迹）
- 关联的目标更新记录
- 关联的成就解锁记录（这次跑步触发了哪些成就）

**操作**：
- 编辑记录（修正数据）
- 删除记录（需二次确认，同时回滚关联的成就和目标进度）
- 标记异常数据（如明显刷数据的记录）

---

#### 4. 目标管理

**目标列表**：
- 表格：用户昵称、目标标题、类型、目标值、周期、当前进度、状态
- 支持筛选：类型（累计/频次/配速/距离）、状态（进行中/已完成/已过期）、周期
- 支持查看目标完成率分布

**系统预设目标模板**：
- 预设一些常见目标供用户一键添加（后续功能）
- 管理预设模板：增删改排序
- 例如："月跑 50km"、"每周跑 3 次"、"5K 破 6"

---

#### 5. 成就系统管理

**成就总览**：
- 20 个成就的解锁统计卡片
- 每个成就显示：名称、分类、稀有度、已解锁人数、解锁率
- 按分类筛选（里程碑/累计/打卡/配速/趣味）
- 按稀有度筛选（普通/稀有/史诗/传说）

**成就详情**：
- 解锁用户列表（谁在什么时候解锁了这个成就）
- 解锁趋势图（按天/周/月看解锁人数变化）

**成就配置**（进阶）：
- 目前成就定义在代码中，后续可以考虑迁移到数据库配置
- 支持动态新增/编辑/下线成就
- 修改成就名称、描述、图标、解锁条件阈值
- 预览成就效果

---

#### 6. 挑战管理

**挑战列表**：
- 表格：发起人、标题、类型、目标值、状态、进度、起止时间
- 支持筛选：状态（进行中/已完成/已失败/已取消）、类型、日期范围

**系统挑战模板**（运营活动）：
- 创建官方挑战活动，所有用户可参加
- 配置：标题、描述、封面图、目标值、时间范围、奖励（专属成就/徽章）
- 管理活动状态：草稿/进行中/已结束
- 查看活动参与人数、完成率

---

#### 7. 内容运营

**公告/通知管理**：
- 发布 App 内公告（如版本更新、活动通知）
- 公告列表：标题、内容、发布时间、状态（草稿/已发布/已下线）
- 支持富文本编辑

**首页 Banner 管理**（后续）：
- 配置首页轮播图
- 设置：图片、跳转链接、排序、展示时间范围

**成就解锁动效配置**（后续）：
- 上传/管理成就徽章的 SVG 图标
- 预览解锁动效

---

#### 8. 系统设置

**管理员账号**：
- 管理员列表
- 新增/编辑/禁用管理员
- 角色权限（超管 / 运营 / 只读，初期可以先不做角色，就一个超管）

**操作日志**：
- 记录管理员的关键操作（删用户、改数据、发公告等）
- 表格：操作人、操作类型、操作内容、时间、IP

**App 配置**：
- GPS 采样频率默认值
- 成就检测开关（可临时关闭某个成就的检测）
- 其他可热更新的配置项

### 后端改动

在现有 Rust/Axum 服务上加一层 admin 路由和权限中间件（`AuthAdmin` Extractor）：

```
新增路由组：
GET  /api/admin/dashboard        仪表盘汇总数据
GET  /api/admin/users            用户列表（分页+搜索）
GET  /api/admin/users/:id        用户详情（含关联数据）
PUT  /api/admin/users/:id        编辑用户
PUT  /api/admin/users/:id/ban    禁用用户
PUT  /api/admin/users/:id/unban  启用用户

GET  /api/admin/runs             全量跑步记录（管理员视角）
GET  /api/admin/runs/:id         记录详情
PUT  /api/admin/runs/:id         编辑记录
DELETE /api/admin/runs/:id       删除记录

GET  /api/admin/goals            全量目标
GET  /api/admin/achievements/stats  成就解锁统计
GET  /api/admin/achievements/:key/unlocks  某成就的解锁用户列表
GET  /api/admin/challenges       全量挑战

POST /api/admin/announcements    发布公告
GET  /api/admin/announcements    公告列表
PUT  /api/admin/announcements/:id  编辑公告
DELETE /api/admin/announcements/:id 删除公告

GET  /api/admin/logs             操作日志
```

新增 Admin 表：
```prisma
model Admin {
  id           String   @id @default(cuid())
  username     String   @unique
  passwordHash String
  nickname     String
  role         String   @default("admin")  // superadmin / admin / viewer
  isActive     Boolean  @default(true)
  lastLoginAt  DateTime?
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt
}

model Announcement {
  id        String   @id @default(cuid())
  title     String
  content   String
  status    String   @default("draft")     // draft / published / archived
  publishAt DateTime?
  expireAt  DateTime?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

model OperationLog {
  id        String   @id @default(cuid())
  adminId   String
  action    String                          // 操作类型
  target    String?                         // 操作对象描述
  detail    Json?                           // 操作详情
  ip        String?
  createdAt DateTime @default(now())
}
```

### 开发优先级

| 优先级 | 模块 | 说明 |
|--------|------|------|
| P0 | 仪表盘 + 用户列表 + 跑步记录列表 | 先把数据能看起来 |
| P1 | 用户详情 + 记录详情 + 成就统计 | 深入看单个用户和成就数据 |
| P2 | 公告管理 + 管理员账号 | 基础运营能力 |
| P3 | 目标模板 + 官方挑战活动 | 运营玩法 |
| P4 | 操作日志 + 角色权限 | 安全和审计 |
